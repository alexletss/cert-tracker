# 🖥️ Локальный деплой без Docker

Этот вариант ставит всё нативно на Windows — без Docker, WSL, Hyper-V, виртуализации.
Подходит когда Docker Desktop не ставится из-за ошибки DISM 1260 или просто не хочется тащить виртуалку.

## Что будет установлено

| Компонент | Что это | Размер |
|---|---|---|
| **PostgreSQL 16** | База данных | ~250 МБ |
| **PostgREST 12** | REST API над базой (один .exe) | 12 МБ |
| **Nginx 1.25** | Веб-сервер для сайта и файлов | 5 МБ |
| _Realtime пропущен_ | _Для 5 пользователей не нужен_ | _0_ |

Всё ставится в `C:\certtracker\`. Память — ~300 МБ суммарно.

## ⚙️ Установка

### Шаг 1. Скачай скрипты

Скачай ZIP репозитория или сделай `git clone`:
```
https://github.com/alexletss/cert-tracker/archive/refs/heads/main.zip
```

Распакуй в `C:\cert-tracker\` (или куда удобно).

### Шаг 2. Запусти установку

**PowerShell от имени админа**, перейди в папку:
```powershell
cd C:\cert-tracker\local-deploy-native
```

Разреши выполнение скриптов (один раз):
```powershell
Set-ExecutionPolicy -Scope Process Bypass -Force
```

Запусти установщик:
```powershell
.\1-install.ps1
```

Что произойдёт:
- Скачает PostgreSQL, PostgREST, nginx (~270 МБ)
- Распакует в `C:\certtracker\`
- Создаст структуру папок: `data\`, `files\`, `logs\`, `web\`
- Создаст `config.env` с настройками

Время: 3–7 минут (зависит от интернета).

### Шаг 3. Настрой пароли

Открой `C:\certtracker\config.env` в Блокноте:
```
notepad C:\certtracker\config.env
```

Замени `POSTGRES_PASSWORD` на свой (любая строка 16+ символов, латиница+цифры).
Остальное можно оставить как есть.

Сохрани.

### Шаг 4. Инициализируй базу данных

```powershell
.\2-init-db.ps1
```

Что произойдёт:
- Создаст системную службу `certtracker-postgres`
- Запустит PostgreSQL
- Создаст базу `certtracker`
- Применит схему из `init-db.sql` (таблицы, индексы, права)

Если всё ок — увидишь `✅ База данных готова`.

### Шаг 5. Запусти все сервисы

```powershell
.\3-start.ps1
```

Что произойдёт:
- Запустит PostgreSQL (если ещё не запущен)
- Запустит PostgREST на порту 3000
- Запустит nginx на порту 80
- Покажет статус всех сервисов

Открой в браузере: http://localhost — должна открыться страница-заглушка «Локальный сервер запущен» с зелёными статусами.

### Шаг 6. Подмени index.html

Скопируй основной сайт в папку nginx:
```powershell
.\4-deploy-site.ps1
```

Этот скрипт:
- Скопирует `index.html` из репозитория
- Заменит `SUPABASE_URL` и `SUPABASE_KEY` на локальные
- Положит в `C:\certtracker\web\`
- Перезапустит nginx

Теперь http://localhost открывает рабочее приложение. Логин: admin / cert2026.

### Шаг 7. Перенеси данные из облака

Открой `local-deploy-native\migration\migrate.html` двойным кликом в браузере. Заполни поля:
- URL локального сервера: `http://localhost`
- Нажми «🚀 Начать перенос»

Дождись «=== ГОТОВО ===». Обнови страницу приложения — увидишь все данные.

### Шаг 8. Открой порт в брандмауэре

Чтобы зайти с других компьютеров в сети:
```powershell
.\5-open-firewall.ps1
```

Это откроет порт 80 в брандмауэре Windows.

Узнай свой IP:
```powershell
ipconfig | findstr IPv4
```

С другого компьютера в сети открой: `http://<IP-сервера>` — например `http://192.168.1.50`.

### Шаг 9. Автозапуск при включении ПК (опционально)

Если хочешь чтобы всё стартовало автоматически при загрузке Windows:
```powershell
.\install-as-service.ps1
```

Скрипт зарегистрирует PostgreSQL, PostgREST и nginx как Windows-службы. Перезагрузи ПК — после загрузки всё уже работает.

## 🛠️ Управление

| Действие | Команда |
|---|---|
| Запустить всё | `.\3-start.ps1` |
| Остановить всё | `.\4-stop.ps1` |
| Проверить статус | `.\status.ps1` |
| Посмотреть логи | `Get-Content C:\certtracker\logs\*.log -Tail 50` |
| Обновить сайт | `.\4-deploy-site.ps1` (после `git pull`) |

## 🗂️ Структура папок после установки

```
C:\certtracker\
├── pgsql\              ← PostgreSQL (программа)
├── data\               ← База данных (файлы Postgres)
├── postgrest\          ← postgrest.exe + конфиг
├── nginx\              ← nginx (программа + конфиг)
├── web\                ← index.html сайта (отсюда раздаётся)
├── files\              ← Загруженные пользователями файлы (PDF, картинки)
├── logs\               ← Логи всех сервисов
└── config.env          ← Настройки (пароли, порты)
```

## 💾 Бэкап базы

Раз в неделю запускай:
```powershell
.\backup.ps1
```

Скрипт создаст `C:\certtracker\backups\certtracker-YYYY-MM-DD.sql` — полный дамп базы. Кидай эти файлы куда-нибудь в облако или на внешний диск.

## ❓ Частые проблемы

| Проблема | Решение |
|---|---|
| `Скрипт не выполняется (политика)` | `Set-ExecutionPolicy -Scope Process Bypass -Force` |
| `Порт 80 занят` | Останови IIS/Skype: `iisreset /stop` или поменяй порт в `nginx\conf
ginx.conf` |
| `Порт 5432 занят` | У тебя уже стоит PostgreSQL — поменяй порт в `config.env` на 5433 |
| `Не открывается с другого ПК` | Запусти `.\5-open-firewall.ps1` или проверь что оба ПК в одной сети |
| `postgres не стартует` | Смотри `C:\certtracker\logs\postgres.log` |
